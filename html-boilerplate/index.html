Links

a - anchor, a link to a different piece of media

href - [ATTRIBUTE] hypertext reference whose value is the link we're directed to

a href="[INSERT LINK HERE]"

target - [ATTRIBUTE] determines whether a new tab will be opened or not. If not, will default to opening in the same tab. Make value "_blank" to open a new tab

target="_blank"

rel - [ATTRIBUTE] describes the relation between the current page and the linked document. "noopener" prevents the opened link from getting access to the original webpage. "noreferrer" prevents the opened link from knowing which webpage has a link to it, includes the noopener behavior so can be used on its own. These are simply for security (e.g. phishing attacks)

rel="noopener noreferrer"


a href="[INSERT LINK HERE]" target="_blank" rel="noopener noreferrer"


Absolute links - links to pages on other external websites

Relative links - links to pages within our own website

When it comes to relative links, a file name is enough. But if the file you want to link to is in a folder/directory within your main folder/directory, you need to write "./[FOLDER]/[FILENAME].[FORMAT]" The number of periods at the start determine how many steps you need to take to reach from the current folder to the folder you want. (e.g. a folder within a folder needs two ../)


a href="./pages/about.html"


An ABSOLUTE link that opens in the same tab
An ABSOLUTE link that opens a new tab
A RELATIVE link that opens a page in the same folder
A RELATIVE link that opens a page in a sub folder

Images

img - displays images

src - [ATTRIBUTE] tells the browser where the image file is located, uses absolute and relative paths too

alt - [ATTRIBUTE] alternative text

width/height - [ATTRIBUTE] determines width and height of image, it's a good habit to still specify even if the size is correct/you're using CSS for it



JPG images - for large color palettes and small file size, good for gradients

GIF images - for simple animations, bad for large color palettes

PNG images - for large image files and transparent pixels

SVG images - for vector-based graphics, good for responsive design, and same uses as PNGs


File Organization

  • Don't use any capital letters
  • Don't use any spaces, use hyphens
  • Keep names as simple and easily-typable as possible because you need to type out file paths in full

How to Write a Good Commit Message

  • Always be specific about what changes you made (provide a subject)
  • Say WHY you made that change
  • Someone that's just joined your programming team should be able to understand fully
  • Describe the problem your commit solves and how
  • Just type git commit without the -m to open up a new window and be able to type multiple lines
  • Commit wheneer there's a meaningful change in code

Introduction to CSS

  • * - universal selector, selects every element
  • Type selector - selects all elements of the given element type
  • Class selector - selects all elements with the given class (an attribute you place on an element class="[NAME]"), these selectors start with a period (.), multiple classes are separated by a space, multiple-worded names are separated by a hyphen
  • ID selector - like a class selector but can only be used once per page and cannot have any spaces, these selectors start with a #, use sparingly
  • Grouping selectors - for when multiple classes share some similar declarations and some unique ones, the similar ones can be grouped in a comma-separated list and the unique declarations can be specified individually
  • Chaining selectors - for when a single element has two classes and you want to give a unique declaration for that specific one, type the classes as a period-separated list, can also do this with IDs if you use a # as a separator
  • Descendant combinator - for when you want to select a specific class within a specific class
  • Create any .css file, usually named "style" or "styles"
  • OR embed internal CSS in the HTML file itself in style tags in the head tag of the file, good for adding unique styles to a single page
  • OR embed inline CSS with the HTML element itself, but it's not recommended (e.g. style="color: white; background-color: black;")
  • Using quotation marks is recommended for fonts with spaces, digits, or punctuation besides hyphens
  • Don't use quotation marks for fonts that are intended as back ups (e.g. sans-serif)
  • Set height or width to 'auto' to make it so that the image retains its original proportions (this is called not "hard-coding" the value)
